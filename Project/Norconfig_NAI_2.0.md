## 4. Project Description

The project aims to develop a new product configurator to support the launch of **NorSeam**, a highly configurable roofing panel product.

### The configurator must:
- Guide the estimator through a controlled configuration path (rules, options, limits)  
- Generate a coherent and usable product specification  
- Produce a client submission integrated with Epicor  
- Support new roofing panel concepts:
  - Front/back  
  - Laps  
  - Notches  
  - Packaging methods  
- Integrate with existing systems:
  - NorConfig  
  - Epicor  
  - Planning systems  
  - planMES  
  - Design tools  
  - ShopFloor / data warehouse  

---

## 5. Rationale (Why)

### Current system limitations:
- Allows configuration outside product offering  
- No validation or limitation rules  
- Monolithic logic (poor UX)  
- Obsolete technologies  
- Heavy reliance on manual input (e.g., cutting lists)  

### Impact:
- Configuration errors  
- External non-conformities  
- Production rework  

---

## 6. Objectives

- Deliver a configurator producing **valid product specifications**  
- Ensure integration with Epicor for **100% of roofing submissions**  
- Prevent invalid configurations  
- Enable easy configuration changes without development  
- Support integration with “take-off” tools  

---

## 7. Priority Justification

The roofing panel is a **strategic growth priority for 2026**.

Without a proper configurator:
- Risk of non-conformities  
- Costly production rework  
- Negative customer experience  

---

## 8. ROI

- Faster and more reliable submissions  
- Reduction of configuration errors  
- Lower production rework costs  
- Reduced workload for design team (via take-off integration)  

---

## 9. Consequences of Not Doing It

- Unstable estimation/configuration process  
- Increased error and non-conformity risks  
- Continued reliance on obsolete system  
- Reduced innovation capability
